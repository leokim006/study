### 내가 선택한 메서드

### 선택 이유
도서관 시스템에서 '대출 가능성'을 가진 모든 객체는 자료의 형태와 상관없이 '대출'과 '반납'이라는 요소를 가지고 있다.
따라서 이를 borrow()와 returnItem()으로 규격화하여 관리의 일관성을 확보하고자 했다.
또한, showInfo()를 통해 각 자료가 가진 고유한 상세 정보를 한 번에 조회할 수 있도록 설계했다.

## 2-2. AI가 만들어 준 뼈대 코드 붙여 넣기
```java
// 공통 규격인 Rentable 인터페이스
public interface Rentable {
    void borrow();       // 대출 기능
    void returnItem();   // 반납 기능
    void showInfo();     // 정보 출력 기능
}

// Rentable 인터페이스를 구현한 Book 클래스
public class Book implements Rentable {
    private String title;
    private boolean isBorrowed;

    public Book(String title) {
        this.title = title;
        this.isBorrowed = false;
    }

    @Override
    public void borrow() {
        if (!isBorrowed) {
            isBorrowed = true;
            System.out.println("종이책 '" + title + "'이(가) 대출되었습니다.");
        } else {
            System.out.println("이미 대출 중인 책입니다.");
        }
    }

    @Override
    public void returnItem() {
        if (isBorrowed) {
            isBorrowed = false;
            System.out.println("종이책 '" + title + "'이(가) 반납되었습니다.");
        } else {
            System.out.println("대출되지 않은 책입니다.");
        }
    }

    @Override
    public void showInfo() {
        String status = isBorrowed ? "대출 중" : "대출 가능";
        System.out.println("[종이책] 제목: " + title + " | 상태: " + status);
    }
}
```

## 3-1. AI 도움 없이 새 클래스 추가하기
이번에는 AI가 만든 `Book`을 참고해서, **직접** `DVD` 또는 `EBook` 클래스를 하나 작성하세요.

public class DVD implements Rentable {
    private String title;
    private boolean isBorrowed;

    public DVD(String title) {
        this.title = title;
        this.isBorrowed = false;
    }

    @Override
    public void borrow() {
        if (!isBorrowed) {
            isBorrowed = true;
            System.out.println("DVD '" + title + "' 대출 완료. (주의: 디스크 스크래치에 주의하세요!)");
        } else {
            System.out.println("해당 DVD는 현재 누군가 빌려갔습니다.");
        }
    }

    @Override
    public void returnItem() {
        if (isBorrowed) {
            isBorrowed = false;
            System.out.println("DVD '" + title + "' 반납 완료. 케이스 확인을 진행합니다.");
        } else {
            System.out.println("이 DVD는 우리 도서관에 있던 게 아니거나 이미 반납되었습니다.");
        }
    }

    @Override
    public void showInfo() {
        String status = isBorrowed ? "대출 중" : "대출 가능";
        System.out.println("[DVD 영상물] 제목: " + title + " | 현재 상태: " + status);
    }
}

## 3-2. 다형성 구조로 묶어 보기
가능하다면 아래와 비슷한 구조를 시도해 보세요.

```java
ArrayList<Rentable> items = new ArrayList<>();
items.add(new Book(...));
items.add(new DVD(...));
```

그리고 반복문으로 모든 자료에 대해 `borrow()` 또는 `showInfo()`를 호출해 보세요.

### 왜 이 구조가 중요한가?
새로운 자료형(EBook, Magazine 등)이 10개, 100개 추가되더라도 자료들을 보관하는 배열의 타입(ArrayList<Rentable>)이나 전체 자료를 순회하며 출력하는 메인 로직의 반복문은 단 한 줄도 수정할 필요가 없기 때문이다.
확장성은 높이면서 유지보수 비용은 획기적으로 줄여주는 객체지향의 핵심 원리라고 볼 수 있다.

## 한 줄 정리
인터페이스의 핵심은 "형태가 다른 객체들에게 공통 규칙을 부여하는 것"이고, 다형성의 핵심은 그 공통 규칙 덕분에 새로운 클래스가 생겨도 기존 코드를 덜 고치게 만드는 것이다.
